## 二、性能优化实现

### 1. 优化版算法服务
**文件**: `src/main/java/com/jinyu/aps/service/AlgorithmServiceOptimized.java`

**优化策略**:

#### 1.1 缓存策略
```java
// 约束检查结果缓存
private final Map<String, Boolean> constraintCheckCache = new ConcurrentHashMap<>();

// 候选机台缓存
private final Map<String, List<MachineLoadContext>> candidateMachineCache = new ConcurrentHashMap<>();
```

**缓存效果**:
- 避免重复计算约束检查
- 预计算候选机台映射
- 缓存命中率统计

#### 1.2 剪枝优化
```java
// 递归深度限制
private static final int MAX_RECURSION_DEPTH = 30;

// 剪枝阈值
private static final int PRUNE_THRESHOLD = 100;

// 动态剪枝检查
if (state.attemptCount > PRUNE_THRESHOLD * tasks.size()) {
    logger.warn("剪枝触发，尝试次数: {}", state.attemptCount);
    return false;
}
```

**剪枝效果**:
- 提前终止无效分支
- 控制递归深度
- 避免无限循环

#### 1.3 并行计算
```java
// 任务预处理并行化
return tasks.parallelStream()
    .map(task -> { ... })
    .sorted(...)
    .collect(Collectors.toList());

// 班次分布并行计算
Map<String, Integer> currentDistribution = details.parallelStream()
    .collect(Collectors.groupingByConcurrent(...));

// 顺位排序并行处理
byMachine.entrySet().parallelStream()

```

**并行效果**:
- 多核CPU利用率提升
- 大数据量处理加速
- 线程安全保证

## 三、测试执行方式

### 3.1 环境要求
- Java 8+
- Maven 3.6+
- H2数据库（测试环境）

### 3.2 执行命令
```bash
# 编译项目
mvn clean compile

# 运行所有测试
mvn test

# 运行特定测试类
mvn test -Dtest=AlgorithmServiceTest
mvn test -Dtest=ScheduleIntegrationTest
mvn test -Dtest=PerformanceTest

# 运行特定测试方法
mvn test -Dtest=AlgorithmServiceTest$AllocationAlgorithmTests#testBasicAllocation_SingleTaskSingleMachine

# 生成测试报告
mvn test -Dmaven.test.failure.ignore=true
```

### 3.3 测试配置
**文件**: `src/test/resources/application-test.properties`

```properties
# H2内存数据库配置
spring.datasource.driver-class-name=org.h2.Driver
spring.datasource.url=jdbc:h2:mem:testdb;DB_CLOSE_DELAY=-1;MODE=MySQL

# 日志配置
logging.level.com.jinyu.aps=DEBUG
logging.level.PERFORMANCE=INFO
```

## 四、性能基准

### 4.1 预期性能指标

| 测试场景 | 任务数 | 机台数 | 预期耗时 |
|----------|--------|--------|----------|
| 小规模 | 10 | 5 | < 500ms |
| 中规模 | 30 | 10 | < 2000ms |
| 大规模 | 50 | 15 | < 10000ms |

### 4.2 约束验证指标

| 约束项 | 限制 | 验证方式 |
|--------|------|----------|
| SKU种类 | ≤4种/机台/日 | 代码断言 |
| 结构切换 | ≤2次/日 | 代码断言 |
| 产能限制 | ≤机台产能 | 代码断言 |
| 班次偏差 | ≤5% | 代码断言 |

|------|--------|------|
